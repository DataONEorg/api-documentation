Use Case 20 - Owner Retrieve Aggregate Logs¶

Revisions

View document revision history.

Goal

Data owners can request aggregated CRUD log for {time period/pid} for all of ‘their’ objects.

Summary

This operation provides a mechanism for data contributors to retrieve use information for content that they own. This is similar to Use Case 19 except that the requesting use is authenticated, and a query on objects owned by the user is used to generate a list of PIDs that indicate log information to be retrieved.

Actors

  • Data owner

  • Coordinating Node

Preconditions

  • User has authenticated

  • Log statistics are available

Triggers

  • Authenticated user requests use information

Post Conditions

  • User has a copy of use information
